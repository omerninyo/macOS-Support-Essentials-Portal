# Chapter 06 - Asset D: Hands-on Lab

## מעבדה: ניהול APFS, פתרון תקלות ב-Spotlight וחקר Clones

### מטרת המעבדה:
ללמד את הסטודנטים לעבוד עם הכלים המובנים של macOS (ממשק משתמש וקצת טרמינל מתקדם) כדי לנתח את חלוקת המקום הדינמית ב-APFS, לחקור את התנהגות ה-Firmlinks, להתמודד עם בעיות מקום פנוי שגוי באמצעות Spotlight, ולהבין הלכה למעשה את הקסם של מנגנון ה-Clones.

### תרחיש:
משתמשת פונה אליכם ומתלוננת: "מחקתי המון קבצים אבל המחשב עדיין אומר שאין לי מקום בגלל System Data עצום, ובנוסף, המחשב בקושי מתפקד כשמעתיקים קבצים גדולים!".

---

### שלב 1: חקר סביבת ה-APFS (הכרת ה-Containers) בממשק הגרפי
1. פתחו את תוכנת ה-**Disk Utility** (מתוך `Applications > Utilities`).
2. בשורת התפריטים העליונה, לחצו על **View** ובחרו **Show All Devices**.
3. נתחו את ההיררכיה בצד שמאל: זהו את הכונן הפיזי (למשל Apple SSD), תחתיו את ה-**Container**, ותחתיו את הכרכים (Volumes).
4. לחצו על ה-**Container**. שימו לב בצד ימין כיצד הוא מציג את סך כל המקום הפנוי שמשותף לכל הכרכים שתחתיו.
5. לחצו על הכרך **Data**. חפשו בחלון המידע את הנתון **Volume Group** (שמראה את שיוכו ל-System) ואת סטטוס ההצפנה (FileVault).
*(טיפ למתקדמים: אם תרצו לצלול לקרביים ולקבל מידע שלא מופיע ב-GUI, פתחו Terminal והריצו `diskutil apfs list`)*.

### שלב 2: יצירה וניהול Volume דינמי ב-APFS
כדי להוכיח שהמקום הפנוי שייך לכולם ללא מחיצות קשיחות:

1. הישארו ב-**Disk Utility** ובחרו בצד שמאל את ה-`Data` Volume שלכם.
2. לחצו על סמל ה-**(+) Volume** בשורת הכלים העליונה (שימו לב לא ללחוץ על סמל ה-Partition!).
3. קראו לו בשם **SharedPool** ולחצו **Add**.
4. שימו לב כמה מהר הוא נוצר (שבריר שנייה) לעומת פרמוט (Partition) מהעבר.
5. לחצו עליו וראו שהמקום הפנוי שלו זהה לחלוטין למקום הפנוי של כונן ה-Data! שניהם חולקים את אותו מאגר.

### שלב 3: חקר מנגנון ההפרדה (Firmlinks)
כדי להבין מדוע המשתמשת לא יכולה למחוק "קבצי מערכת" מסוימים:

1. נסו ליצור קובץ ריק בכונן המערכת דרך הטרמינל (הדרך היחידה לעקוף את מנגנוני ה-GUI ולראות את החסימה האמיתית ברמת הקרנל):
   ```bash
   sudo touch /System/test.txt
   ```
   *(תקבלו שגיאת `Read-only file system` - זוהי ההוכחה שה-SSV חתום וסגור הרמטית)*
2. הריצו את הפקודה המציגה את רשימת ה-Firmlinks המלאה שמגשרת בין המחיצות:
   ```bash
   cat /usr/share/firmlinks
   ```
3. אתרו ברשימה את הנתיב `/Applications`. הבינו שכאשר אתם פותחים את התיקייה הזו, ה-macOS בעצם מערבב תיקייה קריאה-בלבד מה-System עם תיקיית ה-Data כדי ליצור אשליה של מחיצה אחת.

### שלב 4: טיפול באשליות מקום ובעיות חיפוש (בנייה מחדש של Spotlight)
המשתמשת מתלוננת שמקום "System Data" מזייף וקבצים חסרים. הפתרון המומלץ והבטוח ביותר הוא לאפס את האינדקס דרך הממשק הגרפי:

1. פתחו את **System Settings** ועברו ל-**Siri & Spotlight**.
2. גללו למטה ולחצו על כפתור **Spotlight Privacy...**
3. גררו את הכונן הראשי שלכם (Macintosh HD) מחלון ה-Finder לתוך רשימת ה-Privacy, ולחצו **Done**. (פעולה זו אומרת למערכת "אל תאנדקסי יותר את הכונן", ומוחקת בפועל את מסד הנתונים הישן).
4. פתחו שוב את ה-**Spotlight Privacy...**, סמנו את הכונן והסירו אותו באמצעות כפתור המינוס (**-**). לחצו **Done**.
5. כדי לראות שהמערכת עובדת, פתחו את ה-**Activity Monitor**, עברו לטאב ה-CPU וחפשו `md`. אתם תראו את התהליכים `mds_stores` או `mdworker` מזנקים – המחשב בונה עכשיו את מסד הנתונים מחדש ומנקה את אשליית המקום הפנוי!
*(טיפ למתקדמים: בטרמינל, הפקודה `sudo mdutil -E /` עושה את אותו הדבר בדיוק).*

### שלב 5: חקר APFS Clones (אפקט הוואו של ה-Finder)
אחת היכולות המרשימות של APFS היא יצירת Clones. ה-Finder של macOS יודע לעשות זאת אוטומטית מאחורי הקלעים!

1. תחילה, נייצר קובץ "כבד" באמת כדי לראות את האפקט. פתחו טרמינל והריצו את הפקודה הבאה שתיצור קובץ דמה נקי בגודל 1GB על שולחן העבודה:
   ```bash
   mkfile 1g ~/Desktop/MassiveFile.img
   ```
2. סגרו את הטרמינל וגשו לשולחן העבודה שלכם.
3. סמנו את הקובץ `MassiveFile.img` והקישו `Cmd + D` (או קליק ימני ו-Duplicate) כדי לשכפל אותו.
4. שימו לב שהשכפול קרה **בשבריר שנייה** למרות שמדובר בקובץ של 1GB! ה-Finder זיהה שזה אותו כונן והשתמש אוטומטית במנגנון ה-Clone של APFS.
5. **הסתרת הקסם:** סמנו את הקובץ המשוכפל החדש שנוצר והקישו `Cmd + I` (כדי לפתוח את חלון ה-Get Info). בחלון המידע תראו שהוא מדווח על 1GB. אפל מסתירה את מנגנון ה-Clone מהמשתמש הממוצע כדי לא לבלבל אותו (למשל, מה יקרה כשיעתיק אותו לדיסק-און-קי?). 
6. **ההוכחה האמיתית (Wow Effect):** כדי באמת לראות שהשכפול לא לקח לכם 1GB מהכונן, פתחו טרמינל קצר והריצו פקודת שימוש בדיסק (Disk Usage) שמציגה את הקצאת הבלוקים הפיזית האמיתית:
   ```bash
   du -h ~/Desktop/MassiveFile*
   ```
   *אתם תראו שהקובץ המקורי שוקל 1.0G, אבל הקובץ המשוכפל שוקל **0B** (או כמה קילובייטים בודדים)! ה-Clone תופס אפס מקום עד שלא תערכו אותו בפועל.*
7. לסיום, מחקו את שני הקבצים לפח המחזור ורוקנו אותו.
