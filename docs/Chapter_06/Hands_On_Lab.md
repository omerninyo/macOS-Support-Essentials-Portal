# Chapter 06 - Asset D: Hands-on Lab

## מעבדה: אבחון מקום פנוי, diskutil וניהול תקלות חיפוש ב-Spotlight

### מטרת המעבדה:
ללמד את הסטודנטים לעבוד עם כלי אבחון שורת הפקודה (Terminal) כדי לנתח את חלוקת המקום הדינמית ב-APFS, לחקור את התנהגות ה-Firmlinks, ולהתמודד עם בעיות נפוצות של חיפוש ומקום פנוי שגוי באמצעות `mdutil`.

### תרחיש:
משתמשת פונה אליכם ומתלוננת: "מחקתי המון קבצים אבל המחשב עדיין אומר שאין לי מקום בגלל System Data עצום, ובנוסף, כשאני מחפשת ב-Spotlight קובץ טקסט שיצרתי היום, הוא לא מוצא אותו!".

---

### שלב 1: חקר סביבת ה-APFS (הכרת ה-Containers)
1. פתחו את ה-**Terminal**.
2. הקלידו את הפקודה הבאה כדי לראות את כל מערך הכוננים:
   ```bash
   diskutil list
   ```
3. נתחו את הפלט: זהו את ה-`synthesized` Container שלכם. שימו לב כמה כרכים (Volumes) יש בתוכו (למשל `Data`, `System`, `Preboot` וכו').
4. כדי לצלול פנימה לקרביים של APFS, הריצו:
   ```bash
   diskutil apfs list
   ```
5. חפשו את הנתונים הבאים בפלט ורשמו לעצמכם:
   - מהו ה-**Capacity** (הנפח הכולל של ה-Container)?
   - מהו ה-**Volume Group** (שמראה את שיוך ה-System ל-Data)?
   - האם הכרך Data מצפין כרגע (FileVault)?

### שלב 2: יצירה וניהול Volume דינמי ב-APFS
כדי להוכיח שהמקום הפנוי שייך לכולם ללא מחיצות קשיחות:
1. פתחו את תוכנת ה-**Disk Utility** ממסך ה-Applications > Utilities.
2. בחרו בצד שמאל את ה-`Data` Volume שלכם.
3. לחצו על סמל ה-**(+) Volume** בשורת הכלים העליונה.
4. קראו לו בשם **SharedPool** ולחצו **Add**.
5. שימו לב כמה מהר הוא נוצר (שבריר שנייה) לעומת פרמוט (Partition) מהעבר.
6. חזרו לטרמינל, הריצו שוב `diskutil list` וראו שהכרך החדש נוסף אך ה-Container לא שינה את נפחו, והכרך משתמש באותו Free Space.

### שלב 3: חקר מנגנון ההפרדה (Firmlinks)
כדי להבין מדוע המשתמשת לא יכולה למחוק "קבצי מערכת" מסוימים:
1. נסו ליצור קובץ ריק בכונן המערכת:
   ```bash
   sudo touch /System/test.txt
   ```
   *(תקבלו שגיאת `Read-only file system` - זוהי ההוכחה שה-SSV חתום וסגור)*
2. הריצו את הפקודה המציגה את רשימת ה-Firmlinks המלאה שמגשרת בין המחיצות:
   ```bash
   cat /usr/share/firmlinks
   ```
3. אתרו ברשימה את הנתיב `/Applications`. הבינו שכאשר אתם פותחים את התיקייה הזו, ה-macOS בעצם מערבב תיקייה קריאה-בלבד מה-System עם תיקיית ה-Data.

### שלב 4: טיפול באשליות מקום (System Data) ובעיות חיפוש באמצעות Spotlight
המשתמשת מתלוננת שקובץ לא מופיע והמקום הפנוי שגוי.
1. תחילה נבדוק שמנגנון ה-Spotlight עובד:
   ```bash
   sudo mdutil -s /
   ```
   *(הפלט אמור להציג `Indexing enabled`)*
2. צרו קובץ טקסט ניסיוני בתיקיית הבית שלכם כדי לוודא שאינדקס תקין קולט אותו:
   ```bash
   echo "Test File Data" > ~/Desktop/SpotlightTest.txt
   ```
3. חפשו את המילה `SpotlightTest` ב-Spotlight (סמל הזכוכית המגדלת למעלה) או בעזרת הפקודה `mdfind SpotlightTest`.
4. אם ה-Spotlight אצל המשתמשת השתגע או שהמקום הפנוי ב-"About this Mac" מזייף, הפעולה היעילה ביותר היא להשמיד את האינדקס ולבנות אותו מחדש. הריצו את הפקודה הבאה:
   ```bash
   sudo mdutil -E /
   ```
5. מיד לאחר הרצת הפקודה, פתחו את ה-**Activity Monitor**, עברו לטאב ה-CPU וחפשו `md`. אתם אמורים לראות את `mds_stores` או `mdworker` קופצים ומנצלים את ה-CPU (הם בונים כעת את מסד הנתונים מחדש). 
6. (אופציונלי): כדי לבדוק אילו נתונים Spotlight מצליח לקרוא מתוך קובץ ספציפי (שלצורך העניין לא מופיע למשתמשת בחיפוש), הריצו את פקודת האבחון על הקובץ שיצרתם:
   ```bash
   mdimport -t -d3 ~/Desktop/SpotlightTest.txt
   ```
   *(הפלט הארוך יראה לכם איזה `mdimporter` פעל על הקובץ ואת כל שדות המטא-דאטה ששנשאבו לתוך ה-Spotlight)*.
